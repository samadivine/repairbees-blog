Publish a draft article by creating a GitHub PR for review.

Publish the draft at: $ARGUMENTS

## Instructions

You are a publishing agent for https://repairbees.com. Your job is to take a finished draft and create a GitHub PR on the personal brand website repo.

### Step 1: Read the Draft

Read the draft file from `drafts/{slug}.md` (or the path provided in arguments).

Verify the article has:
- [ ] Complete frontmatter (title, description, date, author, category, tags, keywords, image, slug)
- [ ] TL;DR box
- [ ] 6+ H2 sections
- [ ] FAQ section
- [ ] Key takeaways
- [ ] Skool CTAs (mid-article and end)

If anything is missing, flag it and ask before proceeding.

### Step 2: Check for Featured Image

Look for the featured image at `public/images/blog/{slug}.webp`.

If the image doesn't exist, note this — the PR can still be created but the image will need to be added separately.

### Step 3: Create the GitHub PR

Use the `gh` CLI to create the PR on the `samadivine/repairbees-blog` repo:

```bash
# Clone or navigate to the website repo
# Create a feature branch
gh api repos/samadivine/repairbees-blog/git/refs -f ref="refs/heads/content/{slug}" -f sha="$(gh api repos/samadivine/repairbees-blog/git/ref/heads/main -q '.object.sha')"
```

Then commit the article file to `content/blog/{slug}.md` and the image (if it exists) to `public/blog/{slug}.webp`.

Create the PR:
```bash
gh pr create --repo samadivine/repairbees-blog --base main --head content/{slug} --title "New article: {title}" --body "$(cat <<'EOF'
## New Article

**Title**: {title}
**Category**: {category}
**Keywords**: {keywords}
**Reading Time**: {readingTime}

## Pre-Publish Checklist
- [ ] Title is compelling and under 60 chars
- [ ] Meta description is 150-160 chars with stat
- [ ] Content is accurate and well-sourced
- [ ] All links work
- [ ] Featured image looks good
- [ ] CTAs have correct UTM parameters
- [ ] No AI tropes or banned words
- [ ] Preview deployment looks correct

## Preview
Check the Vercel preview deployment for visual review.

---
Generated by Content Engine (Cowork)
EOF
)"
```

### Step 4: Report

Show:
- The PR URL
- The article title and category
- The preview deployment URL (Vercel auto-generates from PR)
- Any warnings or missing elements

Remind the user to:
1. Review the preview deployment
2. Check the article reads well on mobile
3. Merge when ready — Vercel will auto-deploy
